# 디자인 개선 제안: 미국-이란 인프라 피해 보고서

## 현재 디자인 진단

현재 파일은 다크 테마 + 빨강/주황 그라데이션 기반의 "전쟁 보고서" 톤을 갖추고 있다. 기본기는 괜찮지만, 전체적으로 **단조롭고 공격적인 색감**이 지배적이라 긴 문서를 읽을 때 피로감이 크고, 시각적 깊이가 부족하다.

---

## 1. 컬러 팔레트 리파인

### 문제
`--accent-red: #ff4444`가 거의 모든 곳(섹션 보더, 카드, 통계, 네비게이션, 호버)에 쓰이면서 시각적 위계가 무너져 있다. "모든 것이 강조"되면 아무것도 강조되지 않는다.

### 제안
- **주 배경을 순수 블랙(`#0a0a0f`)에서 약간의 색조가 있는 다크 네이비(`#0c0e1a`)로** 바꾸면 차가운 깊이감이 생긴다.
- 빨강 액센트는 **피해 금액, 피해 등급 등 정말 위험한 데이터에만** 한정하고, 나머지 UI 요소(네비게이션, 섹션 제목 보더 등)는 **차분한 중성 톤(슬레이트 블루 `#64748b` 또는 쿨 그레이)**으로 교체.
- 카드별로 **섹터 컬러를 일관되게** 적용: 에너지=앰버, 교통=틸, 군사=레드, 통신=바이올렛, 수자원=시안. 현재 아이콘에만 적용된 이 컬러를 카드 좌측 보더와 헤딩에도 확장.
- `--gradient-fire`를 남용하지 말고, 정말 "불" 관련 컨텐츠(에너지 시설 화재 등)에만 사용.

## 2. 타이포그래피 고급화

### 문제
Noto Sans KR 단일 서체로 전체를 커버하고 있어 밋밋하다. `font-weight: 300`이 본문 기본이라 가독성이 낮다.

### 제안
- **영문 제목용 서체를 고급 세리프로 교체**: `Playfair Display`가 import되어 있지만 섹션 넘버에만 쓰이고 있다. 히어로 타이틀이나 섹션 제목의 영문 부분에도 활용하면 격조가 올라간다. 또는 `Cormorant Garamond`, `Libre Baskerville` 같은 서체 고려.
- **한글 본문 weight를 300에서 400으로** 올리고, `letter-spacing: -0.02em`을 추가해 한글 자간을 자연스럽게 조정.
- 섹션 넘버(`01`, `02`...)의 크기가 5rem으로 과도하게 크다. **3.5rem 정도로 줄이고, opacity를 0.07에서 0.04로** 낮춰서 더 은은한 배경 넘버링으로.
- h3 제목의 `color: var(--accent-orange)` 대신 **밝은 화이트 + 좌측에 섹터별 컬러 도트** 조합으로 교체.

## 3. 레이아웃 & 여백 (Whitespace)

### 문제
카드들이 수직으로 쌓이기만 해서 시각적 리듬이 없다. 모든 섹션이 동일한 구조라 스크롤 피로도가 높다.

### 제안
- **주요 시설 카드는 2컬럼 그리드로** 배치 (데스크톱 기준). 피해 규모가 큰 시설만 풀 와이드 카드로 강조.
- 섹션 간 여백을 `5rem`에서 **`7rem ~ 8rem`으로** 늘리고, 섹션 사이에 **시각적 구분자(divider)** 삽입: 단순한 `border-bottom` 대신 미세한 그라데이션 라인이나 페이드 아웃 효과.
- `content` max-width를 `1000px`에서 **`1100px`**로 살짝 넓히되, 본문 텍스트는 `max-width: 720px`로 제한하여 **최적 가독 폭(60~75자)** 유지.
- 히어로 섹션 하단에서 stats-bar로의 전환을 **부드러운 그라데이션 오버레이**로 처리.

## 4. 카드 디자인 개선

### 문제
모든 카드가 동일한 `damage-card` 스타일. 좌측 4px 빨간 보더가 유일한 시각적 구분인데 12장 이상의 카드에 반복되면 의미가 퇴색된다.

### 제안
- **피해 등급별 카드 스타일 분리**: 
  - 완전 파괴: 상단에 붉은 그라데이션 배너 + "CRITICAL" 뱃지
  - 심각한 손상: 현재 좌측 보더 스타일 유지
  - 경미한 손상: 보더 없이 미세한 배경색 차이만
- 카드 내부의 `.card-grid`를 **라벨-값 쌍이 좌우로 나뉘는 현재 구조에서, 핵심 수치(비용, 생산 영향)만 상단에 크게 표시하는 "히어로 수치" 패턴**으로 변경:
  ```
  ┌─────────────────────────────────┐
  │  사우스 파르스 가스전            │
  │  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
  │  $26억        12% 생산 차단     │  ← 큰 숫자, 볼드
  │  재건 비용     생산 영향         │  ← 작은 라벨
  │                                 │
  │  위치: ...  공격일: ...         │  ← 세부 정보는 아래에 작게
  └─────────────────────────────────┘
  ```
- 카드 hover 시 `translateY(-2px)` 대신 **`box-shadow` 강도만 변경**: 불필요한 위치 이동은 시선을 분산시킨다.

## 5. 데이터 시각화 추가

### 문제
모든 데이터가 텍스트와 테이블로만 전달되고 있어 보고서 특유의 임팩트가 부족하다.

### 제안
- **상단 stats-bar의 숫자들에 카운트업 애니메이션** 추가 (IntersectionObserver 트리거). 순수 CSS 또는 가벼운 JS로 구현 가능.
- 섹션 7(재건 비용 종합)에 **수평 바 차트** 삽입: 각 항목별 비용을 CSS `width` 비율로 표현하는 단순한 막대 그래프. 라이브러리 없이 `div` + `width: calc(...)` 로 충분.
- 걸프 지역 피해 시설 테이블 위에 **간단한 CSS 기반 지도 표현**: 원형 마커를 절대 위치로 배치한 개략적 중동 지도 (SVG). 각 마커에 hover 시 시설명 표시.
- 타임라인 섹션을 현재 `timeline-connector` 스타일에서 **수평 타임라인**으로 변경하면 시간 흐름을 더 직관적으로 보여줄 수 있다.

## 6. 마이크로 인터랙션 & 애니메이션

### 문제
`animate-on-scroll`이 단순한 fadeInUp 하나뿐이라 모든 요소가 같은 방식으로 등장한다.

### 제안
- **스태거 딜레이(stagger delay)**: 같은 섹션 내 카드들이 순차적으로 등장하도록 `transition-delay`를 0.1s씩 증가. CSS만으로 가능: `.animate-on-scroll:nth-child(2) { transition-delay: 0.1s; }` 등.
- 통계 숫자에 **숫자 카운트업**: `0`에서 목표치까지 증가하는 애니메이션.
- 네비게이션의 **active 상태 전환을 부드럽게**: 현재 JS로 직접 `style.color`를 조작하고 있는데, 대신 `.active` 클래스를 토글하고 CSS `transition`으로 처리.
- 카드 내 비용 수치에 **미세한 글로우(glow) 효과**: `text-shadow: 0 0 20px rgba(255,68,68,0.3)` — 붉은 숫자가 살짝 빛나는 느낌.

## 7. 히어로 섹션 업그레이드

### 문제
히어로가 텍스트와 미세한 SVG 패턴 배경뿐이라 첫인상이 밋밋하다.

### 제안
- **반투명 위성사진 또는 추상적 지도 이미지를 배경에 깔기**: `background-image`에 어두운 overlay를 씌운 위성/지도 이미지. 무료 소스로 NASA Worldview 위성 이미지 활용 가능.
- 또는 **CSS로 구현하는 동적 파티클/그리드 배경**: `radial-gradient`를 여러 레이어로 겹쳐서 미세한 움직임 효과. `@keyframes`로 배경 위치를 천천히 이동.
- 히어로 뱃지의 `pulse` 애니메이션이 약간 산만하다. **초기 1회만 발동 후 정지**하거나, pulse 대신 **정적인 글로우 보더**로 교체.
- 스크롤 인디케이터를 단순 텍스트("SCROLL") 대신 **얇은 화살표 아이콘 + 미세한 바운스**로.

## 8. 테이블 디자인

### 문제
테이블이 기능적이지만 시각적으로 평범하다.

### 제안
- **행 간 배경색 교차(stripe)**: `tr:nth-child(even) td { background: rgba(255,255,255,0.015); }` — 미세하지만 행 구분이 훨씬 쉬워진다.
- 테이블 헤더에 **하단 그라데이션 보더** 추가: `border-image: linear-gradient(to right, var(--accent-orange), transparent) 1`.
- 피해 내용 셀이 길어질 때를 대비해 **`max-width`와 말줄임 또는 확장 토글** 고려.
- 모바일에서 테이블을 **카드 형태로 리플로우**: `@media (max-width: 768px)` 에서 각 행을 세로 카드로 변환.

## 9. 네비게이션 개선

### 문제
수평 스크롤 네비게이션이 모바일에서 발견성이 낮고, 현재 위치 표시가 약하다.

### 제안
- active 링크에 **하단 보더 대신 배경 하이라이트**: `background: rgba(255,68,68,0.08); border-radius: 6px;` — 더 눈에 띄는 active 표시.
- **스크롤 진행률 표시줄**: 네비게이션 바 하단에 1~2px 높이의 프로그레스 바를 추가하여 전체 문서에서 현재 위치를 시각적으로 보여준다.
- 모바일에서 **스크롤 가능하다는 힌트**: 네비게이션 우측에 미세한 페이드 그라데이션 오버레이 + 화살표.

## 10. 다크 모드 내 깊이감 (Elevation System)

### 문제
`--bg-primary`, `--bg-secondary`, `--bg-card` 3단계만 있어 z축 깊이감이 평면적이다.

### 제안
- **5단계 elevation 시스템** 도입:
  - Level 0: `#0c0e1a` (페이지 배경)
  - Level 1: `#11132a` (섹션 배경)
  - Level 2: `#181b38` (카드)
  - Level 3: `#1f2345` (카드 hover / 팝업)
  - Level 4: `#262a52` (모달 / 오버레이)
- 각 레벨에 **미세한 보더 밝기 차이**를 두어 요소 간 분리감 강화.
- `backdrop-filter: blur(20px)`가 네비게이션에만 적용되어 있는데, **비용 요약 카드나 disclaimer 같은 중요 요소에도 글래스모피즘** 적용 고려.

---

## 11. 인쇄 & 접근성

- `@media print` 스타일 추가: 배경색 제거, 텍스트 블랙, 링크에 URL 표시.
- `prefers-reduced-motion` 미디어 쿼리로 모든 애니메이션 비활성화 옵션.
- 색상 대비 비율 점검: `--text-secondary: #9b9b9b`는 다크 배경 위에서 WCAG AA 기준(4.5:1)을 간신히 통과. `#a8a8a8` 이상으로 밝히는 것을 권장.
- 테이블에 `scope="col"` / `scope="row"` 속성 추가.
